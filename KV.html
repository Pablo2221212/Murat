1. Что такое «система контроля версий»?

Система контроля версий (Version Control System, VCS) — это программное обеспечение, которое отслеживает изменения в файлах и папках с течением времени

2. К какому типу VSC относится? В чем главное отличие от других типов?

Git относится к распределенным системам контроля версий (Distributed Version Control System, DVCS).

Главное отличие DVCS от других типов (например, централизованных) заключается в том, что каждый разработчик имеет полную копию репозитория (включая всю историю изменений) на своем локальном компьютере.

3. В чем заключается преимущество использование Git?

Преимущества использования Git вытекают из того, что это распределенная система контроля версий, а также из его особенностей и функциональности

4. Какие команды Git вы знаете?

git init: Инициализирует новый локальный репозиторий Git.

git config --list

git branch

git add .

git commit -m "message"

git push -u origin main

git remote add origin

5. В чем отличие Git и Git Hub?

Git: Это система контроля версий. Это программное обеспечение, которое вы устанавливаете на свой компьютер и используете для отслеживания изменений в своих проектах. Git работает локально, на вашем компьютере.

GitHub: Это веб-платформа, которая предоставляет хостинг для Git-репозиториев в облаке. Это онлайн-сервис, где вы можете хранить свои Git-репозитории, совместно работать над проектами, отслеживать проблемы, выполнять код-ревью и многое другое. GitHub предоставляет веб-интерфейс и API для работы с Git-репозиториями.
